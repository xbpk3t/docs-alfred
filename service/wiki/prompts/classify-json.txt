## wikiType 枚举
- repo_eval: 对开源项目/代码仓库的评测、分析、推荐
- deep_dive: 对某个技术主题的深入分析、教程、原理讲解
- inbox: 信息不足或需要人工判断

## 分类原则
- 如果 URL 是 GitHub/GitLab 等代码仓库，或正文明显在评测/介绍某个开源项目，优先使用 repo_eval。
- 如果正文是可读的文章、问答、教程、研究博客或视频/播客 transcript，并且候选目录中存在明确匹配主题，选择最合适的候选 path。
- 不要为了提高归档率而选择泛化、牵强或仅有弱关联的 topic；候选不明确时返回 topicPath="none"、wikiType="inbox"、needsManualReview=true。
- needsManualReview=true 用于正文不可用、明显是错误页/登录页/导航页，或者没有明确匹配候选目录。
- topicPath 必须从候选 path 中逐字选择；不要创造新 path。
- summary 应基于正文内容，不要复述网页标题或候选目录。
- summary 必须是合法 JSON string；不要把 Markdown 编号写成 `\1.`、`\2.`、`\3.` 这类非法 JSON 转义。

## 输出要求
只返回一个 JSON object，不要 Markdown code fence，不要额外解释。字段如下：

{
  "topicPath": "候选 path 或 none",
  "wikiType": "repo_eval|deep_dive|inbox",
  "contentType": "text|video|audio",
  "summary": "Markdown 中文摘要，包含 概述、关键要点、值得关注 三个小节",
  "confidence": 0.0,
  "needsManualReview": false
}

如果正文是错误页、登录页、空壳页面、纯导航、广告页、无法判断主题或候选都完全不匹配，返回 topicPath="none"、wikiType="inbox"、needsManualReview=true。
