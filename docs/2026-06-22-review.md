### MF-2. 库代码使用 `log.Printf` — 违反项目规则

**`pwgen/pkg/generator.go:104,110,117,144,151`**

项目 CLAUDE.md 明确要求 "日志使用 `log/slog`；除 CLI 用户输出外，不在 package 中随意 `fmt.Println`"。此处 5 处使用 `log.Printf`/`log.Println`，且在库代码（非 cmd/）中。

### MF-3. goroutine 泄漏 — signal handler 无退出机制

**`data-cli/cmd/enrich.go:99-102`**

```go
go func() {
    <-sigCh
    cancel()
}()
```

函数正常返回时（未收到信号），此 goroutine 永远阻塞在 `<-sigCh`。应 select `ctx.Done()`：

```go
go func() {
    select {
    case <-sigCh:
        cancel()
    case <-ctx.Done():
    }
}()
```

### MF-4. 无界 map 内存泄漏

**`service/wiki/write.go:55-58`**

```go
var pathLocks = struct {
    locks map[string]*sync.Mutex
    mu    sync.Mutex
}{locks: make(map[string]*sync.Mutex)}
```

每个被锁定的路径添加一个 entry，永远不清理。长时间运行的进程或测试套件中持续增长。
